[문제상황]

증상: Admin PC는 HQ1에는 SSH 접속되지만 HQ2는 접속 실패(또는 그 반대)


[원인 후보]

HQ2만 SSH/VTY/ACL 설정 누락 또는 불일치

HQ2로의 라우팅/경로 문제(특정 인터페이스 down, OSPF 누락)

접속 대상 IP 선택 오류

[확인]

1. HQ1/HQ2 설정 비교

장비: HQ1/HQ2

명령: show run | section ip ssh

명령: show run | section line vty

명령: show access-lists 10

2. 경로 확인

장비: Admin PC

명령: ping <10.255.0.2>

기대: ping 성공

[해결]

HQ2에 SSH/VTY/ACL을 HQ1과 동일 정책으로 정렬

라우팅/인터페이스 복구 후 재시도


[재검증]

Admin PC에서 HQ1/HQ2 모두 SSH 접속 성공 확인

show ssh로 세션 확인
