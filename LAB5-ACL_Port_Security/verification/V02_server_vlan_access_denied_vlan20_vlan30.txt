[검증항목]

항목: VLAN 20 / VLAN 30 -> Server VLAN 접근 차단 여부

목적: 서버 보호 정책에 따라 VLAN20/30은 서버 VLAN(50) 접근이 제한되는지 확인

기대결과:

VLAN20 -> 서버(192.168.50.3) 접근 실패(차단)

VLAN30 -> 서버(192.168.50.3) 접근 실패(차단)


[구성 참고]

Allow_Intra_OUT에서 VLAN20/30에 대해 host 단위 permit 또는 deny 정책 확인

ACL 적용 인터페이스와 방향(out/in) 확인

[실행]

1. VLAN20 PC에서 서버로 ping

PC: <VLAN20_PC>

명령: ping <192.168.50.3>

2. VLAN30 PC에서 서버로 ping

PC: <VLAN30_PC>

명령: ping <192.168.50.3>
