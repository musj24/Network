# TS04 - 외부 통신은 되는데 특정 트래픽(ICMP/HTTP)이 안 됨

## 증상
- ping은 되는데 웹(HTTP)이 안 되거나, 반대로 웹은 되는데 ping이 안 됨
- 외부 도메인 해석은 되는데 실제 접속이 실패

## 유력 원인
- PT 환경에서 특정 트래픽(특히 ICMP/HTTP) 처리/inspection 제약
- 방화벽/경계 장비의 기본 정책 차이(시뮬레이터 특성)

## 확인 절차
1) ICMP와 HTTP를 분리해서 테스트
- ping 8.8.8.8
- 웹 브라우저로 외부 웹 접속
2) 경계 장비(ASA 사용 시) inspection/정책 확인(가능한 범위)
- inspect icmp/http 유무 확인

## 조치
- 검증 시 트래픽을 2종 이상(ICMP + HTTP)로 수행해 “한쪽만 되는 현상”을 분리 진단
- PT 제한이 의심되면 검증 근거를 시뮬레이션 캡처로 남기고 README에 제약 명시

## 재검증
- ICMP/HTTP 각각의 성공 여부가 설계 의도와 일치
