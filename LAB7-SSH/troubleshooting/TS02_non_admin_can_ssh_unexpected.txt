[문제상황]

증상: 비인가 PC가 HQ1/HQ2로 SSH 접속이 가능함


[원인 후보]

VTY에 access-class 10 in이 적용되지 않음(또는 다른 라인에 적용)

ACL 10이 과도하게 허용

VTY 라인 범위 누락

장비별( HQ1/HQ2 ) 설정 불일치

[확인]

1. VTY 라인 설정 확인

장비: HQ1/HQ2

명령: show run | section line vty

기대:0 4 VTY 라인에 access-class 10 in 적용

2. ACL 10 범위 확인

장비: HQ1/HQ2

명령: show access-lists 10

기대: host 192.168.50.5만 permit

3. 실제 접속 세션 확인

장비: HQ1/HQ2

명령: show ssh / show users

기대: 비인가 세션 존재 여부 확인

[해결]

ACL 10을 host permit만 남기고 재구성

VTY 전체 범위(환경에 따라 vty 0 4 / 0 15)를 일관되게 적용

transport input ssh로 Telnet 등 불필요 입력 제거

[재검증]

비인가 PC에서 SSH 재시도(실패해야 정상)

Admin PC에서 SSH 접속 성공 유지 확인
